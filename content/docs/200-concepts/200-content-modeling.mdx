Contentlayer 的主要工作是将内容转换为可以导入到 web 项目页面中的数据。为了有效地做到这一点，Contentlayer 必须知道内容的形状—— _内容模式_.

这将根据您使用的是本地源（文件）还是远程源而有所不同。

## 远程源

<Callout>远程资源是实验性的。预计本节会有所更改。</Callout>

对于远程源（即无头 CMS），内容模式是从源的 API 自动引入的。

然而，我们仍在试验远程源的确切工作方式。随着我们继续探索和添加新的源，我们将更新本节。

## 本地来源

当使用本地源时，必须显式定义模式，以便 Contentlayer 知道如何处理本地文件。

您可以通过[浏览文件源代码指南](/docs/sources/files)了解更多关于本地源代码的信息.

下面，您将发现一些在本地建模内容时需要理解的重要概念。

### 文档与嵌套

有两种类型的对象——*文档*和*嵌套（文档）*。

**文档**是 Contentlayer 根据*文档类型定义*生成的对象。将文档类型想象成数据库中的模型或表。文档就是该模型的一个实例，即数据库查询的结果。

Contentlayer 将文档写入单独的 JSON 文件，然后这些文件可以导入到项目的页面中。

**嵌套**文档是直接嵌入文档中的对象。它们被定义为*嵌套类型*，主要用于可重复的形状。

例如，假设您有一个 SEO 对象，您希望它包含在各种模型中。您可以定义嵌套文档类型一次，然后在任何需要它的模型中使用它。 [`defineNestedType` API 参考](/docs/reference/source-files/define-nested-type)有更多信息和示例。

### 内容参考

就像在数据库中一样，内容可以与其他内容相关联。例如，发布文档类型可以引用标记文档。

然而，[引用当前未转换或嵌入](/docs/reference/source-files/field-types#reference). 它们作为相关对象的文件路径引用传递。使用引用时，我们建议构建实用程序函数，以便根据需要检索相关数据。
