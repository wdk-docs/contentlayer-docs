您可以通过多种方式参与进来，帮助 Contentlayer 成为在前端应用程序中处理内容的最佳方式：

- 使用 Contentlayer
- 提供支持
- 提升 Contentlayer
- 建筑示例
- 编写文档
- 贡献代码

## 使用 Contentlayer

对您来说，为 Contentlayer 做出贡献最简单、最有价值的方法是**使用它、提供反馈和报告错误**

如需反馈和一般对话，请[加入我们的 Discord 社区](https://discord.gg/rytFErsARm). 要报告错误，请[在 GitHub 上打开一个问题](https://github.com/contentlayerdev/contentlayer/issues/new).

## 提供支持

随着 Contentlayer 使用量的增长，我们将需要帮助来跟上[GitHub 上的问题](https://github.com/contentlayerdev/contentlayer/issues). 我们欢迎社区成员帮助确认错误报告，同时也将他们的用例和视角添加到功能请求中。

## 提升 Contentlayer

如果你喜欢 Contentlayer，告诉全世界！发推特，写一篇博客。在下一个网络项目的计划会议上提出它。不管是什么，你谈论的内容层越多，内容层就变得越好。

## 建筑示例

我们现在有一组小而集中的例子。我们的目标是扩展这些内容，但要深思熟虑。太多的例子可能会分散用户的注意力，而构建者则很难维护。

如果你想建立一个新的例子，或者认为你有一个有用的例子，请联系。在 Discord 上的#contribution 中发送消息，或直接发送消息`seancdavis29#9241`，我们会为您找到合适的示例。

## 编写文档

医生们有一个良好的开端，但还有很长的路要走。如果您有兴趣编写文档，请在 Discord 上 DM`seancdavis29#9241`。

## 贡献代码

如果您对贡献代码感兴趣，请[打开一个问题](https://github.com/contentlayerdev/contentlayer/issues/new)或者找到你想解决的问题。添加一条评论，说明你想解决这个问题，我们将一起制定计划，因为代码库很复杂。
