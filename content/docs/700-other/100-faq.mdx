## Contentlayer 正在解决什么问题？

现代 web 框架没有规定解析内容的方法。它们提供了强大的页面路由和呈现过程，但提供内容取决于您。

这是一项艰巨的工作。如果没有时间和精力来构建一个真正出色的处理机制，很容易迅速降低这些工具为我们提供的开发人员体验。

Contentlayer 通过简化 web 项目中的内容处理，持久保持了现代 web 框架提供的出色开发人员体验。

## 为什么 Contentlayer 速度很快？

Contentlayer 充分利用构建工具的优化，使处理源内容变得轻而易举。

然后，它智能地缓存内容，并逐步构建。当您更新内容时，Contentlayer 将利用已经完成的工作，只构建已经更改的内容。

## 我可以将 Contentlayer 与现有工具一起使用吗？

内容层被构建为与框架无关。Contentlayer 的核心是一个内容处理器，但它提供了用于从各种来源导入内容的模块，并使用插件提供与现代框架的紧密集成。

我们的文档列出了[当前支持的来源](/docs/sources)和[框架](/docs/environments). 如果你看不到你在寻找什么，[开始讨论](https://github.com/contentlayerdev/contentlayer/issues/new). 我们一直在寻找新的用例来考虑。

## Next.js 已经支持 MDX。为什么我需要 Contentlayer？

Next.js 天生支持 MDX，这是一种误解。Next.js 不会对您的内容所在的位置以及应该如何处理进行任何推断。

像[mdx-bundler](https://github.com/kentcdodds/mdx-bundler)这样的库和[next-mdx-remote](https://github.com/hashicorp/next-mdx-remote)提供工具来帮助处理 MDX。除了处理 MDX，Contentlayer 还解析和验证内容，并提供自动生成的 TypeScript 类型定义。它在优先考虑性能和开发人员体验的同时也做到了这一点。

了解有关[Contentlayer 与其他内容处理程序的比较](/docs/concepts/comparison#content-processors)的更多信息.
